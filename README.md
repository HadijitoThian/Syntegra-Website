# Syntegra Corporate Website

The marketing site for **Syntegra** — an AI Solutions company building practical AI tools for Indonesian SMEs.

Live products linked from this site:

- Syntegra Social Media Builder — https://syntegra-landing-builder-production.up.railway.app/
- Syntegra POS — https://www.pos.syntegra.co.id/
- Syntia AI Assistant — https://www.syntia.syntegra.co.id/

## Stack

- Next.js 14 (App Router) + TypeScript
- Tailwind CSS + `@tailwindcss/typography`
- `next-intl` v4 for ID (default) / EN
- Framer Motion (subtle animations)
- MDX blog via `@next/mdx` + `next-mdx-remote/rsc` + `gray-matter`
- Claude Haiku 4.5 via `@anthropic-ai/sdk` for lead qualification

## Getting started

```bash
cp .env.example .env.local
# fill in ANTHROPIC_API_KEY and NEXT_PUBLIC_WHATSAPP_NUMBER
npm install
npm run dev
```

Open http://localhost:3000 — you'll be redirected to `/id`.

## Environment variables

| Var | Required | Purpose |
|---|---|---|
| `ANTHROPIC_API_KEY` | yes (form) | Claude Haiku for `/api/qualify-lead` |
| `NEXT_PUBLIC_WHATSAPP_NUMBER` | yes | Floating WhatsApp button number (no `+`, e.g. `62818718388`) |
| `NEXT_PUBLIC_SITE_URL` | optional | Used for sitemap / canonical URLs |
| `DEEPINFRA_API_KEY` | only when regenerating images | Used by `scripts/generate-images.mjs` (Flux 1.1 Pro) |

## Regenerating images

Hero / product / team / blog images live in `public/generated/` and were
generated by Flux 1.1 Pro via DeepInfra. To regenerate (e.g. after editing
prompts in `scripts/generate-images.mjs`):

```bash
node scripts/generate-images.mjs
```

Each image costs ~$0.04. The script writes JPEGs directly into
`public/generated/`. Commit the files so production doesn't need the API key.

## Deploy to Vercel

1. Push this repo to GitHub.
2. Import the project on Vercel — zero-config works.
3. Add the 3 env vars in the Vercel dashboard.
4. Set the custom domain (e.g. `syntegra.co.id`) under **Settings → Domains**.

## File structure

```
app/
  [locale]/          # ID + EN — all marketing pages
    page.tsx         # home
    produk/          # products overview + 3 detail pages
    tentang/         # about
    insights/        # blog index + [slug] post
    kontak/          # contact
  api/qualify-lead/  # POST endpoint → Claude Haiku
  sitemap.ts
  robots.ts
components/
  layout/            # Navbar, Footer, LanguageToggle, WhatsAppFloat
  sections/          # Hero, ProductsShowcase, etc.
  ContactForm.tsx    # The AI-powered lead form
content/insights/    # MDX blog posts, organized by locale
i18n/                # next-intl setup + ID/EN message JSON
lib/                 # anthropic.ts, mdx.ts, utils.ts
```

## Updating content

| What | Where |
|---|---|
| Copy / translations | `i18n/messages/id.json` and `i18n/messages/en.json` |
| Product feature lists | `app/[locale]/produk/{slug}/page.tsx` (features array) |
| WhatsApp number | `NEXT_PUBLIC_WHATSAPP_NUMBER` env var |
| Blog posts | drop a new `.mdx` file under `content/insights/{id|en}/` |
| "Future Products" placeholders | `components/sections/FuturePipeline.tsx` |
| Logo | replace `/public/logo.svg` (currently a text gradient placeholder) |
| Product mockups | drop PNGs at `public/products/{social-media-builder,pos,syntia}.png` |

## Notes

- The default locale is `id`. Visiting `/` redirects to `/id`.
- The blog renders MDX server-side via `next-mdx-remote/rsc`.
- `/api/qualify-lead` falls back to a generic friendly message if the Anthropic call fails — the user always sees a success screen, but errors are logged server-side.
- Lead submissions are currently `console.log`ged only — see the TODO in `app/api/qualify-lead/route.ts` for wiring up storage (Supabase, Notion, email, etc.).
